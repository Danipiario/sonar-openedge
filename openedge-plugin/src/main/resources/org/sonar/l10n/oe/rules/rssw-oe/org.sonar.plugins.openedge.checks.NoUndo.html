This rule raises an issue whenever it finds a variable, temp-table, property or parameter definition without the NO-UNDO attribute. Any object without this attribute will keep a before-image copy in transactions, leading to I/O in the LBI file. While in some cases you may want to use the UNDO feature on variables, it's in most cases useless.

Noncompliant Code Example

  DEFINE VARIABLE myVar AS CHARACTER.

Compliant Solution

  DEFINE VARIABLE myVar AS CHARACTER NO-UNDO.
